## Issues Discussed (from prior sessions, documented here for completeness)

### 1. UOM Conversion Factor Error
- **When:** Submitting Tailoring Measurement with extras
- **Cause:** `conversion_factor` not set when appending items to submitted SO
- **Fix:** Added `conversion_factor: 1, uom: "Nos", stock_uom: "Nos"` to item dict
- **File:** `tailoring_measurement.py:_add_extras_to_sales_order()`

### 2. Commission Calculation TypeError
- **When:** Saving SO after extras added
- **Cause:** `base_rate`, `base_amount`, `net_rate`, `net_amount`, `base_net_rate`, `base_net_amount` were None
- **Fix:** Set all 6 monetary fields explicitly when appending items
- **File:** `tailoring_measurement.py:_add_extras_to_sales_order()`

### 3. Number Cards Blank on Workspace
- **When:** Loading Asafat Tailoring workspace
- **Cause:** `show_percentage_stats` enabled on Number Cards with custom field filters (Frappe bug)
- **Fix:** Disabled `show_percentage_stats` on all 4 cards via bench console
- **File:** `install.py:_setup_number_cards()`

### 4. SO Totals Not Including Extras
- **When:** After measurement submit adds extras to SO
- **Cause:** `so.save()` alone doesn't recalculate totals on submitted docs
- **Fix:** Added `so.calculate_taxes_and_totals()` before `so.save()` in both add and remove functions
- **File:** `tailoring_measurement.py` (both `_add_extras_to_sales_order` and `_remove_extras_from_sales_order`)

### 5. Extras Rate Showing 0.00
- **When:** Extra items with amount but no rate
- **Cause:** Rate not being derived from amount/qty
- **Fix:** Added rate derivation: `if rate == 0 and amount > 0: rate = amount / qty`
- **File:** `tailoring_measurement.py:_add_extras_to_sales_order()`

### 6. Advance Not Updating on PE Cancel
- **When:** Cancelling Payment Entry linked to SO
- **Cause:** No hook existed for PE cancellation
- **Fix:** Created `events/payment_entry.py` with SQL-based recalculation on both submit and cancel
- **File:** `events/payment_entry.py`, `hooks.py`

### 7. Unwanted Create Options on SO
- **When:** Viewing submitted Sales Order
- **Cause:** ERPNext core adds standard Create menu options
- **Fix:** `setTimeout` + `frm.page.remove_inner_button()` for 6 unwanted options
- **File:** `public/js/sales_order.js`

### 8. Sales Return Feature
- **When:** Customer returns delivered items
- **Implementation:** Added `process_sales_return()` API + `show_return_dialog()` JS function
- **Creates:** Credit Note (return SI) + optional refund PE
- **Files:** `api.py`, `public/js/sales_order.js`

## Decisions Made

1. **UOM for Extras:** Keep "Nos" as standard UOM for Extras item group
2. **Advance tracking:** Use SQL-based recalculation (sum all submitted PEs) rather than increment/decrement
3. **Number Cards:** Disable percentage stats to work around Frappe bug
4. **Totals recalculation:** Always call `calculate_taxes_and_totals()` before save on submitted docs
5. **Extras removal:** Remove from end of SO items list in reverse (assumes extras are always appended at end)
6. **Revenue fallback:** Use SO grand_total if no submitted Sales Invoices exist
