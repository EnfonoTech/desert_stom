Desert Stom — Documentation

Custom ERPNext v15 / Frappe v15 app that layers a complete tailoring workflow on top of standard Selling — measurements, per-cloth specs, advance collection, stitching status, completion billing, returns, and profit tracking.

Frappe v15 ERPNext v15 1 Module · 3 DocTypes 6 API methods 2 Reports · 4 Print Formats

Overview

Desert Stom turns a standard ERPNext Sales Order into a tailoring job ticket. Each order carries a Stitching Status, linked Tailoring Measurement cards (with per-garment body measurements stored as JSON), advance payments, and live profit/loss reference figures. Completion creates Sales Invoice, Payment Entry, and Delivery Note in one step; returns create credit notes and optional refunds.

Order Workflow

Stitching status drives the whole lifecycle. Buttons appear on the submitted Sales Order based on the current status.

Order→ Measurement→ Job Order→ Processing→ Process Completed→ Ready for Delivery→ Delivered

Installation

cd /home/v15/frappe-bench
bench get-app https://github.com/EnfonoTech/desert_stom --branch master
bench --site designer-stom.enfonoerp.com install-app desert_stom
bench --site designer-stom.enfonoerp.com migrate
bench build --app desert_stom
bench --site designer-stom.enfonoerp.com clear-cache
bench restart

after_install creates custom fields, search-field property setters, the Extras item group, enables the UOM stock setting, and builds 4 dashboard number cards. Print formats and default-print-format property setters arrive via fixtures.

DocTypes

Tailoring Measurement Submittable

Main measurement card. Naming MEAS-.YYYY.-.#####. Holds header, cloth details, body measurements, collar/neck/hip/button options, an extras child table, and a hidden measurements_json that stores per-cloth measurement sets switched by the cloth_name field.

Controller logic: on_submit upgrades SO Order → Measurement and appends extra items to the Sales Order (with full UOM/rate fields + calculate_taxes_and_totals()). on_cancel reverses both.

Measurement Item Child

One row per garment carried from the Sales Order: item_code, item_name, qty, sales_order_item.

Measurement Extra Item Child

Add-on items appended to the SO on submit: item_code, item_name, qty, rate, amount (auto-calculated).
